How to Upload Your Website to GitHub

A comprehensive step-by-step guide to hosting your website with GitHub Pages

1

Create a GitHub Account

If you don't have one already, go to github.com and sign up for a free account. Choose a username that you'll be comfortable sharing professionally.

Pro Tip

Your username will be part of your GitHub Pages URL (username.github.io), so choose wisely!

2

Install Git on Your Computer

Download and install Git from the official website:

https://git-scm.com/downloads

After installation, verify it's working by opening a terminal/command prompt and typing:

git --version

Configure Git with your name and email (use the same email you used for GitHub):

git config --global user.name "Your Name" git config --global user.email "your.email@example.com"
3

Create a New Repository

On GitHub, click the "+" icon in the top right and select "New repository".

Name your repository username.github.io (replace "username" with your GitHub username). This is important for GitHub Pages.

Make sure to set the repository to "Public" and initialize it with a README.

Important

For your first GitHub Pages site, the repository must be named exactly as your username followed by .github.io

4

Clone the Repository to Your Computer

On your repository page, click the "Code" button and copy the URL.

Open a terminal/command prompt, navigate to where you want to store your project, and type:

git clone https://github.com/cleverfox-77/cleverfox-77.github.io.git

Replace the URL with your actual repository URL.

5

Add Your Website Files

Copy all your website files (HTML, CSS, JS, images, etc.) into the cloned repository folder.

Make sure your main page is named index.html as this is the default file GitHub Pages will look for.

File Structure

Your repository should have an index.html file at the root. Other files can be organized in folders like css/, js/, images/, etc.

6

Commit and Push Your Files

In the terminal, navigate to your repository folder and run these commands:

git add . # Stages all your files for commit git commit -m "Initial commit - adding my website" # Commits the files with a message git push origin main # Pushes the files to GitHub

You may be prompted to enter your GitHub username and password (or personal access token).

7

Enable GitHub Pages

Go to your repository on GitHub, click on "Settings", then select "Pages" from the left sidebar.

Under "Source", select the "main" branch and click "Save".

Note

If your repository is named correctly (username.github.io), GitHub Pages will be automatically enabled, but it's good to check the settings.

8

Access Your Website

Your website will now be live at:

https://cleverfox-77.github.io

It might take a few minutes for your site to be available after the first deployment.

Important Note

If you want to use a custom domain instead of username.github.io, you can configure it in the GitHub Pages settings. You'll also need to add a CNAME file to your repository and configure DNS settings with your domain registrar.

Practice with This Sample Website

You can use the HTML code of this page as your first website to upload to GitHub. Just right-click on the page, select "View Page Source", copy the entire code, and paste it into an index.html file in your repository.

Alternatively, you can create your own simple HTML file:

<!DOCTYPE html>
<html>
<head>
  <title>My First GitHub Website</title>
  <style>
    body { font-family: Arial, sans-serif; text-align: center; padding: 50px; }
    h1 { color: #28a745; }
  </style>
</head>
<body>
  <h1>Hello World!</h1>
  <p>My website is hosted on GitHub Pages!</p>
  <p>Visit my GitHub profile: <a href="https://github.com/cleverfox-77">cleverfox-77</a></p>
</body>
</html>
